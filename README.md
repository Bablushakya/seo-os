# SEO-OS — Internal SEO Operations Operating System

Internal off-page SEO operations platform for the India Heritage Travel team (Bharat, Bablu, Rahul). Built with Next.js 14 App Router, TypeScript strict mode, Tailwind CSS, Shadcn UI, and Supabase.

---

## Supabase Auth Configuration (AUTH-001-01)

Before launching the application, configure your Supabase project with the following security and authentication settings:

1. **Disable Public Signups**:
   - Go to **Supabase Dashboard** -> **Authentication** -> **Providers** -> **Email**.
   - Set **Allow new users to sign up** to **OFF** (Disabled). This prevents public registrations. Only pre-invited/provisioned team members can access the app.

2. **Disable Email Confirmation**:
   - In the same **Email** provider configuration section, set **Confirm email** to **OFF** (Disabled).
   - This allows the internal users created via the provisioning script to log in immediately with their temporary credentials without needing email verification.

3. **Set Minimum Password Length**:
   - Go to **Authentication** -> **Policies** (or Auth Settings in newer Dashboards).
   - Under **Password complexity rules**, set the minimum password length to **12** characters.

4. **Set Session Duration**:
   - Go to **Authentication** -> **Sessions** (or Session Settings).
   - Set **JWT Expiry** to **604800 seconds** (7 days) to match the technical specifications for session persistence (allows long-lasting refresh tokens).

## Initial Setup & Migrations

### 1. Database Migrations
1. Open the [Supabase Dashboard](https://supabase.com/dashboard).
2. Go to the **SQL Editor** on the left menu.
3. Open a new query tab.
4. Copy and paste the contents of `supabase/migrations/001_initial_schema.sql` into the editor.
5. In **Part 7: Sync Auth.Users with Public.Users** of the SQL file, verify or update the email trigger check list to match your team's real emails (it defaults to `bharat@indiaheritage.com`, `bablu@indiaheritage.com`, and `rahul@indiaheritage.com`).
6. Click **Run** to execute the script and provision the 18 tables, triggers, enums, indexes, and Row-Level Security (RLS) policies.

### 2. Configure Environment Variables
Copy `.env.example` to `.env.local` in the project root:
```bash
cp .env.example .env.local
```
Fill in the following credentials from your Supabase Project Settings (Settings -> API):
- `NEXT_PUBLIC_SUPABASE_URL`
- `NEXT_PUBLIC_SUPABASE_ANON_KEY`
- `SUPABASE_SERVICE_ROLE_KEY` (Required for the backend admin operations and provisioning script)

## Account Provisioning (AUTH-001-07)

To provision the three default team accounts:
1. Ensure your `.env.local` contains a valid `SUPABASE_SERVICE_ROLE_KEY`.
2. Run the provisioning script from the project root:
   ```bash
   npx ts-node scratch/provision-users.ts
   ```
3. Copy the secure temporary credentials generated by the script and distribute them to the respective team members:
   - **Bharat** (admin)
   - **Bablu** (seo_specialist)
   - **Rahul** (data_specialist)

## Local Development

1. Install dependencies:
   ```bash
   pnpm install
   ```
2. Start the development server:
   ```bash
   pnpm dev
   ```
3. Open [http://localhost:3000](http://localhost:3000) to log in.
